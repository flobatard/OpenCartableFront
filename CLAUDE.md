# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Projet

Front Angular 22 d'**OpenCartable**, plateforme pédagogique auto-hébergée (AGPLv3) : un prof authentifié (OIDC/Zitadel) compose des cours, les élèves les consultent via liens publics sans compte. Cadrage produit : `Descriptions.md`. Design system (source de vérité UI) : `DESIGN_SYSTEM.md`.

## Commandes

Node est géré par nvm ; dans un shell non-interactif, npm n'est pas sur le PATH :

```bash
export PATH="$HOME/.nvm/versions/node/v26.3.0/bin:$PATH"
```

```bash
npm start                              # dev server (4200)
npm run build                          # build prod + prerender de la home
npm test                               # vitest (jsdom)
npm test -- --include src/app/core/theme/theme.service.spec.ts   # un seul fichier
npm test -- --filter "ThemeService"    # par nom de suite/test
npm run serve:ssr:OpenCartableFront    # sert dist/ via Express (4000)
docker compose up --build              # conteneur SSR (4000)
```

Après tout changement touchant SSR/i18n/prerender, vérifier que `dist/OpenCartableFront/browser/index.html` contient bien le texte français rendu (pas de chaînes vides) : c'est le symptôme d'une régression du préchargement Transloco.

## Architecture et invariants

**Zoneless** (pas de zone.js — défaut Angular v22) : tout état partagé est exposé en **signaux** ; les souscriptions rxjs écrivent dans des signaux. Ne jamais ajouter `provideZoneChangeDetection`.

**Trois modes de rendu** (`src/app/app.routes.server.ts`) : `/` prerendered au build, `/auth/callback` client-only (il manipule URL et storage), `**` SSR. Toute nouvelle route doit choisir son mode ; une route qui touche `window`/`localStorage` sans garde ne peut pas être SSR/prerender. Une route protégée par `authGuard` doit être en `RenderMode.Client` (le guard renvoie `false` au serveur — jamais authentifiée au SSR, donc jamais prerendable) : cf. `:lang/subjects`.

**Couche données HTTP** : `SubjectService` (`src/app/core/subjects/`) est le patron des accès API. Un seul `HttpClient.get` par ressource, mis en cache via `shareReplay({ bufferSize: 1, refCount: false })`, état exposé en signaux (`tree`/`loading`/`error`) alimentés par la souscription rxjs. Le Bearer Zitadel est attaché **automatiquement** par l'intercepteur d'`angular-oauth2-oidc` à toute requête sous `environment.apiUrl` (`resourceServer.allowedUrls` dans `app.config.ts`) — ne pas ajouter de header manuel ; `apiUrl` inclut déjà `/api`. Fetch déclenché côté navigateur uniquement (guard `isPlatformBrowser`). Les helpers purs de l'arbre (recherche, chemin d'ancêtres, aplatissement, lignes de treeview) vivent dans `subject.utils.ts`, testés isolément. `EducationLevelService` (`src/app/core/education-levels/`) est le second exemplaire du patron (mêmes signaux + cache) ; ses helpers minimaux vivent dans `education-level.utils.ts` — on ne généralise **pas** `subject.utils.ts` pour ne pas coupler les deux pickers. Les libellés métier (`nom` des matières et des niveaux d'étude) viennent de l'API et **ne passent pas** par l'i18n.

`UserProfileService` (`src/app/core/users/`) est la variante **mutable** du patron : pas de `shareReplay` figé mais un signal `profile` source de vérité, une **promesse en vol partagée** (`ensureLoaded()` — callback et guard n'émettent qu'un GET, invalidée sur erreur pour permettre le retry), les mutations remplacent le signal (`saveProfile()` : le `PUT /users/me/profile` sert d'onboarding initial **et** d'édition de profil, sémantique remplacement, pas de refetch) et un `effect` purge le profil quand `AuthService.isAuthenticated()` retombe. La logique du formulaire de profil (forme du FormGroup typé, règles de cohérence, mapping payload, complétude) est **factorisée en helpers purs** dans `core/users/profile-form.ts`, partagés par l'onboarding et la page profil — les templates, eux, restent distincts (stepper vs page à plat). Piège d'ordre : `patchFormFromProfile` pose `systeme` avant les blocs, sinon la cohérence câblée viderait les niveaux patchés.

**Espace cours** (`core/courses/` + `features/courses/`) : `CourseService` est la variante mutable **multi-ressources** du patron — signaux `list`/`detail` sources de vérité **refetchés à chaque entrée de page** (pas de `shareReplay` : données mutables), mutations async (`createCourse`, `addBlock` — qui porte un méta titre/description optionnel —, `deleteBlock`, `reorderBlocks`, `updateBlockContent`, `updateBlockMeta`, `updateBlockResource` — PATCH dédié `{resource_id}` des blocs `document`, jamais mêlé au content ni au méta) qui patchent localement le signal `detail` (depuis la réponse du back, **sauf `reorderBlocks` qui est optimiste** : il réordonne le signal **avant** le PUT et laisse l'appelant recharger sur erreur ; `#patchDetail` ne s'applique que si le détail affiché est le cours muté), purge à la déconnexion. **Quatre types de blocs, tous créables** : `texte`, `exercice`, `document` (pont nullable vers une ressource de la bibliothèque — `resource_id` en colonne ; supprimer la ressource supprime le bloc, FK CASCADE côté back) et `module` (placeholder J4, éditeur = notice `unsupported`). Les blocs portent `titre`/`description` facultatifs (tous types, distincts du `content`). Helpers purs du formulaire de création dans `core/courses/course-form.ts` (motif `profile-form.ts`) ; ceux du méta titre/description d'un bloc dans `core/courses/block-meta-form.ts`, **partagés par la modale de création et l'éditeur** ; ceux du contenu document (`buildDocumentForm`, `payloadFromDocumentContent` — normalisation avec replis —, `payloadFromDocumentForm`) dans `core/courses/document-form.ts`. Pages : liste `courses` (cartes, badges matières/niveaux résolus dans les arbres — id inconnu = pas de chip), création `courses/new` (**déclarée avant** `courses/:id`, matching en ordre ; picker de niveaux filtré par le `systeme_scolaire` du profil), page cours `courses/:id` à **trois onglets Blocs | Ressources | Aperçu** (tablist APG au motif `markdown-field`, ids ARIA par compteur de module ; panneaux en `@if` — pas de Monaco ici, rien à préserver ; **cyclage clavier ←/→ par index via `TAB_ORDER`** ; onglet actif reflété dans `?tab=` — Blocs = défaut donc param retiré, `resources`/`preview` sérialisés — via `router.navigate([], {replaceUrl: true})`, deep-linkable ; le retour d'éditeur retombe sur Blocs) — onglet Blocs : liste = titre en gras + description ; **création via une modale** `block-create-dialog` saisissant titre/description, puis **redirection** vers l'éditeur du bloc créé ; « Modifier » sur **tous** les types ; **réordonnancement par glisser-déposer (poignée `grip-vertical` + `cdkDragHandle`, `@angular/cdk/drag-drop`) ou flèches ↑/↓ discrètes — poignée aussi opérable au clavier (↑/↓/Début/Fin), via helper pur `moveIdTo` ; le câblage DnD ne binde pas `cdkDropListData` ni n'appelle `moveItemInArray` (le signal `detail` reste seule source de vérité)** ; suppression en deux temps sans modale, désarmée au blur — onglet Ressources : composant `CourseResources` (voir ci-dessous) dont l'output `(deleted)` fait recharger le détail du cours par la page (les blocs document pointeurs ont été supprimés en cascade par le serveur) — onglet Aperçu : composant `CoursePreview` (voir ci-dessous), **vue élève** du cours entier —, éditeur de bloc `courses/:id/blocks/:blockId`. Toutes protégées `[authGuard, onboardingGuard]` + `RenderMode.Client`, params lus en snapshot (pas de `withComponentInputBinding` dans ce projet). Le header porte le lien « Mes cours » quand la session est ouverte.

**Bibliothèque de ressources** (`core/resources/` + `features/courses/course-resources/`) : `ResourceService` est un exemplaire de plus du patron mutable (signaux `list`/`listLoading`/`listError` refetchés à chaque entrée d'onglet, garde `#courseId` sur les patchs locaux, purge à la déconnexion) plus un signal `uploadState` (`idle|presigning|uploading|confirming|error` + `progress`, un upload à la fois). **Upload en trois temps** : POST presign (type dérivé du MIME par `resourceTypeFromMime`, repli `application/octet-stream`) → **PUT direct navigateur→S3 via `HttpClient`** — hors `environment.apiUrl`, donc l'intercepteur OIDC n'attache **pas** de Bearer (`allowedUrls`), c'est voulu ; ⚠ le header `Content-Type` doit être **strictement** le mime déclaré au presign (figé dans la signature S3) ; `reportProgress` alimente la progression — → POST confirm **sans body** (la ressource devient `disponible`, aucun bloc créé). Helpers purs dans `core/resources/resource.utils.ts` (`resourceTypeFromMime`, `formatBytes` — formatage maison déterministe, pas d'`Intl`). `CourseResources` (présentationnel + service) : liste (nom, badge type, taille, date, `en_attente` atténuée — bordure dashed, texte `--text-secondary`, téléchargement désactivé), upload par `<input type="file" hidden>` (input vidé après choix pour permettre le retry), **renommage inline** (FormControl public `renameControl`, Échap annule ; seul le nom affiché change, la clé S3 reste figée), téléchargement (`getDownloadUrl` puis `window.open(url, '_blank', 'noopener')` dans le composant), suppression en deux temps désarmée au blur. Clés i18n sous `courses.resources.*` et `courses.tabs.*`.

**Aperçu global du cours** (`features/courses/course-preview/`) : `CoursePreview` (`app-course-preview`, onglet Aperçu) rend **tous les blocs prévisualisables les uns à la suite des autres, dans l'ordre du back** — le cours tel que le verra l'élève. **Vue élève** : blocs `module` **omis** et réponses attendues des exercices **masquées** (via `exerciseMarkdownFromContent`, cf. `exercise-form.ts`). Rendu **par bloc** (pas de markdown concaténé : les documents s'intercalent et ne sont pas du markdown) : `texte` (`content.markdown`) et `exercice` (sujet + énoncés) passent par `app-markdown-view` ; `document` par `CoursePreviewDocument` (`app-course-preview-document`) — **média intégré** (image/audio/vidéo affichés en ligne via l'URL présignée `getDownloadUrl`, résolue au montage dans un `effect`) avec **repli carte téléchargeable** pour les PDF/autres (`type: 'document'`) et sur échec de présignature ; ressource supprimée/inconnue → notice `documentMissing`. Présentational + services (`CourseService.detail` déjà chargé par la page hôte, `ResourceService.loadList` déclenché au montage — couvre le deep-link `?tab=preview` à froid). Les deux `app-markdown-view` (texte/exercice) reçoivent `[courseId]` pour résoudre les ressources `oc-resource:` intégrées au markdown (cf. **Rendu markdown de cours**) — indépendant du bloc `document`, résolu par `CoursePreviewDocument`. Client-only (`markdown-view` et présignature touchent `window`). Clés i18n sous `courses.preview.*` et `courses.tabs.preview`.

**Éditeur Monaco** (`shared/markdown-editor/`, CVA `string` enrobant `ngx-monaco-editor-v2`) — pièges vérifiés dans le source du wrapper, ne pas « simplifier » :
- Monaco est **servi en AMD depuis les assets** (`angular.json` copie `node_modules/monaco-editor/min/vs` → `/monaco/vs`, ~16 Mo), jamais bundlé ; `NGX_MONACO_EDITOR_CONFIG` est fourni par le composant avec un **baseUrl absolu `/monaco/vs`** (le défaut du wrapper est relatif à la page → cassé sur les routes profondes). Workers et CSS monaco s'auto-configurent depuis ce chemin.
- Le wrapper **dispose et recrée l'éditeur à chaque changement de référence de `[options]`** → options figées en constante module ; le thème passe par `monaco.editor.setTheme` global (effect sur `ThemeService.theme()`), **jamais** par les options.
- Le wrapper **n'est pas SSR-safe** (touche `window`/`document` sans guard) → toute page hôte en `RenderMode.Client` + `@if (isBrowser)` dans le composant.
- Relais CVA avec **guard anti-écho** (le wrapper ré-émet la même valeur après un `writeValue`) ; `remeasureFonts()` quand JetBrains Mono finit de charger (sinon curseur décalé).
- En jsdom (specs), le loader AMD ne se charge jamais → le wrapper est **inerte par construction** : les specs pilotent les `FormControl`, jamais monaco.
- Méthode publique **`insertAtCursor(text)`** (pour l'insertion de ressource, cf. `markdown-field`) : capte l'instance `IStandaloneCodeEditor` émise par `(onInit)` (auparavant jetée), `executeEdits` + `focus`. La mutation du modèle **passe la garde anti-écho et se propage seule** au contrôle hôte — pas de `#onChange` manuel ; **no-op** tant que monaco n'est pas initialisé (SSR/jsdom).

**Vue markdown réutilisable** (`shared/markdown-view/`, `app-markdown-view`) : composant présentational en lecture seule qui rend une chaîne markdown en HTML sûr via le pipeline de `course-markdown` (markdown+KaTeX synchrone puis passe Mermaid async, `effect` avec stale-guard `onCleanup`, re-rendu sur `ThemeService.theme()`, bypass DOMPurify). **C'est l'extraction du pipeline d'aperçu**, jadis dupliqué à l'identique dans `markdown-field` et `exercise-editor` ; il rend dès qu'il est **monté** (le montage est gouverné par le `@if` de l'hôte, seule la garde `isPlatformBrowser` subsiste). Consommé par `markdown-field`, l'aperçu complet d'`exercice` et `course-preview` — trois sites. Client-only. Input **optionnel** `courseId` : quand il est fourni **et** que le markdown référence des ressources, une **troisième passe** `resolveCourseResources` est chaînée sous le **même stale-guard** (après Mermaid) pour résoudre les `oc-resource:` — le composant injecte alors `ResourceService`, lit `list()` réactivement (re-rendu à l'arrivée de la bibliothèque), présigne via `getDownloadUrl`, et **ne résout qu'une fois `listLoading()` retombé** (évite un flash « indisponible »). Un `effect` de **chargement défensif** (`untracked`, une fois par cours) ne charge la liste **que si aucun hôte ne l'a fait** (liste vide et pas en cours — sinon il écraserait la liste déjà présente) : en pratique ce sont les pages hôtes (`block-editor`, `course-preview`) qui chargent.

**Champ markdown réutilisable** (`shared/markdown-field/`, CVA `string`) : composite qui embarque les **onglets Éditeur | Aperçu** (tablist APG), l'`app-markdown-editor` (Monaco), l'**aperçu** (délégué à `app-markdown-view`, alimenté par le signal `draft` de frappe, sous `@if` sur l'onglet aperçu) et la modale d'aide. C'est l'unité que composent les éditeurs de contenu — bloc texte (une instance) et éditeur d'exercice (plusieurs : sujet + un énoncé par question ; la réponse attendue reste un textarea simple, décision produit). Composition d'un `FormControl` interne (patron `SubjectMultiPicker`) exposé **public** `control` (exception à la convention `protected` : jsdom ne peut pas taper dans monaco, les specs pilotent le contrôle) ; panneau éditeur masqué par **`[hidden]`, jamais `@if`** (un `@if` rechargerait monaco à chaque bascule) ; ids ARIA du tablist **uniques par instance** via compteur de module (**jamais** `Date.now()`/`Math.random()`). Usage **Client-only** (rendu DOMPurify + Monaco touchent `window`). `MarkdownHelpDialog` (`shared/markdown-help-dialog/`) est la modale d'aide Markdown/LaTeX/Mermaid extraite : `<dialog>` natif présentational, méthodes publiques `open()`/`close()`, ref `#dialogEl` (**pas** `#dialog` : une ref de template ne doit pas collisionner avec le signal `viewChild` `dialog`). Onglets et pilules d'action sont des **utilitaires globaux** (`.tabs`/`.tab`/`.tab--active`, `.pill`/`.pill--active`) dans `_components.scss`, à côté de `.btn` (cf. `DESIGN_SYSTEM.md §8`). Les clés i18n de ces composants partagés vivent sous `markdownField.*` / `markdownHelp.*` (et **non** `courses.editor.*`). **Insertion de ressource** : input `courseId` (injecte `ResourceService`) ; en contexte cours (`canInsert` = `courseId` non vide), un bouton « Insérer une ressource » (pilule, à côté de l'aide) ouvre `ResourcePickerDialog` (`shared/resource-picker-dialog/`, nouveau — présentational `<dialog>` natif au motif `MarkdownHelpDialog`, `open()`/`close()`, `#dialogEl`, output `(pick)`) listant les ressources `disponible` de la bibliothèque ; le choix construit le snippet (`buildResourceMarkdown` : `![nom](oc-resource:id)` pour une image, `[nom](…)` sinon) et l'**insère au curseur** via `editorRef().insertAtCursor(...)` — la modale ferme d'abord (le focus revient au déclencheur) puis émet, l'insertion rendant ensuite le focus à Monaco. `courseId` est aussi descendu à `app-markdown-view` (résolution d'aperçu). Clés `markdownField.insertResource`/`resourceMissing`, `resourcePicker.*`.

**Éditeur de bloc** (`features/courses/block-editor/`) : **coquille-page** (en-tête, **formulaire titre/description à enregistrement explicite — tous types**, indicateur d'autosave du contenu, espace de travail redimensionnable éditeur/assistant par poignée `separator` ARIA + pointer capture, états de chargement). Le **méta** (titre/description) est un `buildBlockMetaForm` séparé, initialisé une fois depuis le bloc, bouton « Enregistrer » actif seulement si **modifié** (snapshot JSON du payload, motif page profil) ; `saveMeta` appelle `updateBlockMeta` qui n'envoie **que** le méta (jamais le `content`), pas de flush au `ngOnDestroy` (save explicite, non autosave). Le **contenu** est délégué par type : `texte` → composite `MarkdownField` (`<app-markdown-field [formControl]="content" [courseId]="courseId">`), `exercice` → `ExerciseEditor` (`features/courses/exercise-editor/`, présentationnel, reçoit `[courseId]` — descendu à **chaque** `app-markdown-field` sujet/questions et à l'aperçu `app-markdown-view` pour l'insertion/résolution de ressources : **trois onglets Sujet | Questions | Aperçu complet** — tablist APG au motif `markdown-field`, cyclage clavier par `TAB_ORDER`, ids ARIA par compteur de module ; panneaux Sujet/Questions masqués par `[hidden]` jamais `@if` (Monaco vit dans les deux), panneau Aperçu en `@if` (pas de Monaco) déléguant à `app-markdown-view` le markdown concaténé `fullExerciseMarkdown` (réponses attendues exclues) — ; `[initial]` lu une seule fois, `(contentChange)` remonte chaque frappe ; **réordonnancement des questions par glisser-déposer (poignée `grip-vertical` + `cdkDragHandle`) ou clavier via `moveQuestionTo`, qui réutilise l'instance de FormGroup — invariants de `moveQuestion` préservés (ids stables J2, `@for track group`, `openGroup` par instance, snapshot `applyGeneratedIds`) — avec `*cdkDragPreview`/`*cdkDragPlaceholder` compacts pour ne pas cloner Monaco** ; helpers purs `core/courses/exercise-form.ts`). Les blocs `document` ont une **section simple** (pas d'espace redimensionnable ni de chat — pas de Monaco) : `DocumentEditor` (`features/courses/document-editor/`, présentationnel, motif `ExerciseEditor` minimal) — picker de ressource = **`<select>` natif** (liste plate courte, pas de treeview) alimenté par les ressources `disponible` du cours (`ResourceService.loadList` déclenché une fois par un effect pour **tout bloc à contenu éditable** — document, mais aussi texte/exercice, dont le `markdown-field` a besoin pour le picker d'insertion et la résolution d'aperçu ; l'hôte charge, les enfants lisent le singleton), `[initial]` lu une seule fois, `(contentChange)` (légende/affichage) rejoint le pipeline d'autosave, mais `(resourcePick)` déclenche un **PATCH immédiat** `updateBlockResource` (une sélection est discrète, pas une frappe) avec état d'erreur dédié et **revert du select** (`resetResource`) sur échec ; un `resourceId` absent de la liste (ressource supprimée) retombe sur l'option vide. Le type `module` tombe dans la notice `unsupported` (placeholder J4). **Autosave** conservé ici, dans un **pipeline unique** : `merge(content.valueChanges, frappes exercice, frappes document) → debounceTime(1500) → concatMap` (**jamais `switchMap`** : les promesses de PATCH ne sont pas annulables, une réponse périmée écraserait la plus récente), signal `saveState` (`idle|dirty|saving|saved|error` — le flux survit aux erreurs, retentative à la frappe suivante), init **une seule fois** depuis le bloc chargé et jamais réécrit depuis le signal (le patch du détail post-save ne doit pas écraser la frappe), flush fire-and-forget au `ngOnDestroy`. **Deux invariants exercice** : le payload est construit **à l'envoi du PATCH** (état courant du formulaire), jamais à l'émission, et les ids de questions générés par le back sont **réécrits après chaque save** (`applyGeneratedIds` sur un snapshot des groupes capturé à l'envoi) — sinon l'autosave suivant renverrait `id: null` et le back régénérerait des ids censés être stables à vie (contrat J2). Le contrôle `content`, le `form` de l'ExerciseEditor et le `form`/`resourceControl` du DocumentEditor sont publics (exception à la convention `protected`) : jsdom ne peut pas taper dans monaco, les specs les pilotent.

**Panneau assistant & espace de travail redimensionnable** (`features/courses/course-chat/` + layout du block-editor) : `CourseChat` (`app-course-chat`) est la **coquille présentationnelle placeholder** de l'assistant IA — en-tête + pastille « bientôt », zone de messages `role="log"` en état vide, saisie **désactivée** ; tokens du DS, clés i18n **`courseChat.*`**, **aucun** HTTP/oauth. Le câblage IA futur (service + signal `messages`/`send()`) est **additif** : les inputs `courseId`/`blockId` sont déjà passés (réservés) et l'output `collapse` remonte le repli au parent. Le block-editor la monte dans un **espace de travail flex à deux colonnes** (`app-markdown-field` | poignée | `app-course-chat`) — **premier layout à colonnes et premier breakpoint (`@media` 900px) du projet**. Invariants à préserver : (1) la largeur de la colonne éditeur vient du signal `editorPct` (borné 15–85 %) via la **custom property `--editor-basis`**, **jamais** un `flex-basis` inline (sinon les états CSS ne peuvent plus la surcharger) ; (2) le **repli** (`chatCollapsed`) masque chat + poignée par **`[hidden]`, jamais `@if`** (préserver la future conversation) et `.block-editor__workspace--solo` fait **grandir** l'éditeur (`flex:1`) à leur place ; (3) la poignée redimensionne via `setPointerCapture` (monaco ne vole pas les events pendant le glissé), axe dérivé du `flex-direction` réel (row → X, column → Y), clavier flèches/Home/End. Le plafond du shell `.app-main` est **relevé (1800px) pour cette seule page** (les autres pages gardent leur propre max-width ≤ 880). Sous **900px** : empilement vertical, **drag conservé** (axe Y, hauteur définie maintenue — l'IA peut être verbeuse) et un **bouton de réouverture** réapparaît sous l'éditeur (`display:none` sur desktop, où la bascule « Assistant » de la barre d'outils suffit) pour rouvrir le chat sans remonter. Libellés des contrôles sous `courses.editor.{resize,toggleChat,showChat}`.

**Rendu markdown de cours** (`core/markdown/course-markdown.ts`) : `renderCourseMarkdown` = instance `Marked` dédiée (ne **jamais** muter le singleton `marked`, son `use()` est global) + extension math maison (`$…$` en ligne, `$$…$$` centré — KaTeX `throwOnError: false`, LaTeX invalide rendu en `.katex-error`) + **DOMPurify** (profils html+mathMl+svg, `ADD_TAGS: semantics/annotation` — jamais `annotation-xml`, vecteur mXSS). **C'est l'unique point de sanitisation du HTML de cours** ; le bypass Angular est, lui, centralisé dans `app-markdown-view` (`shared/markdown-view/`), seul consommateur qui injecte cette sortie via `bypassSecurityTrustHtml` — justifié car le sanitizer d'Angular dépouillerait les `style` inline et le MathML/SVG de KaTeX — et ne bypasse jamais autre chose ; `markdown-field`, l'aperçu d'exercice et `course-preview` passent tous par lui. ⚠ Sans `window` (SSR), DOMPurify retourne le HTML **non filtré** : consommation Client-only obligatoire. `katex.min.css` est chargé globalement (angular.json, fontes copiées au build) ; `katex`/`dompurify` sont des dépendances directes, alignées sur les versions transitives de mermaid.

**Ressources de la bibliothèque intégrées au markdown** (`core/markdown/course-resource-ref.ts` — schéma **partagé par le rendu ET l'insertion**) : le markdown stocke une **référence stable** `oc-resource:<id>` (l'`id` de `CourseResource`, **jamais** l'URL présignée — TTL court, elle expirerait). Deux temps : **(1)** l'override des renderers `image`/`link` de `courseMarked` transforme un href `oc-resource:<id>` en placeholder `<img>`/`<a data-oc-resource-id>` **sans src/href** (l'attribut `data-*` survit à DOMPurify par défaut ; aucune requête réseau prématurée ; tout autre href retombe sur le rendu marked par défaut via `return false` — vérifié : `this.parser.parseInline(tokens)` pour le texte du lien) ; **(2)** `resolveCourseResources` (passe **async navigateur**, patron `renderCourseDiagrams` : DOMParser → `replaceWith` → re-sanitise) remplace chaque placeholder par l'élément choisi **selon le `type` réel** de la ressource — **image en ligne, audio/vidéo en lecteurs intégrés, PDF/autres en lien téléchargeable** —, l'URL présignée fraîche étant résolue à la volée par un callback `resolve` (une présignature par id distinct, même répété) ; ressource supprimée/`en_attente`/injoignable → note `course-resource--missing`. La re-sanitisation ajoute `ADD_ATTR: ['target','aria-label']` (le reste — `controls`/`loading`/`src`/`href`/`alt`/`class` — est déjà autorisé par défaut). `hasCourseResources` garde l'entrée dans la passe (comme `hasCourseDiagrams`). Cette approche DOM (vs. remplacer la chaîne markdown brute) respecte la structure : un `oc-resource:` dans un bloc de code reste du texte, jamais résolu. Helpers purs `parseResourceRef`/`resourceRefHref`/`resourceKind`/`buildResourceMarkdown` (échappe `[`/`]` du nom).

**Extensions markdown** (`shared/markdown-extensions/` — **tout** y vit : interface, registry, passe placeholder, parseurs, composants) : un fence d'un « langage » custom (```` ```geogebra ````, ```` ```jsxgraph ````) est rendu par un **composant Angular monté dynamiquement** — premier usage de `createComponent`/`afterRenderEffect` du repo. Chaque langage = une `MarkdownExtensionDef` (`language`, `isPrintable`, `loadComponent` **lazy — jamais d'import statique du composant depuis la def**, sinon il entre dans le bundle initial via app.config) enregistrée en multi-provider (`provideMarkdownExtensions()` dans `app.config.ts`) ; `MarkdownExtensionRegistry` indexe et **mémoïse les imports** (un échec est retiré du cache → retentative possible). **Deux temps** : **(1)** passe placeholder **synchrone pure** (`applyExtensionPlaceholders`, patron `renderCourseDiagrams`) dans l'effect de rendu de `markdown-view`, entre `renderCourseMarkdown` et le premier `#html.set` — chaque `pre > code.language-<lang>` **enregistré** devient `<div class="course-extension course-extension--pending" data-oc-extension data-oc-printable>` avec la source du fence en textContent (survit aux re-sanitize ; langage non enregistré = bloc de code normal ; **pas de re-DOMPurify** : nœuds construits par API DOM, l'invariant « LA sanitisation vit dans core/markdown » tient) ; **(2)** un `afterRenderEffect` lisant `html()` monte les composants **après chaque application du `[innerHTML]`** : `createComponent({hostElement: placeholder})` + `setInput('source', …)` + `appRef.attachView` — son `onCleanup` pose le stale-flag (couvre l'`await` d'import) et **détruit les ComponentRef du cycle précédent** (chaque set d'`html` souffle les hôtes) ; `textContent` vidé et `--pending` (typo mono du repli) retiré **seulement après import réussi** (repli source façon mermaid). **Impression** : `transformForPrint` (print.service, signature objet `PrintLabels`) substitue chaque `[data-oc-extension]` dont `data-oc-printable ≠ "true"` par la note `.oc-print__extension-note` (« contenu interactif ») — posé par la passe placeholder depuis la def, donc valable même non monté ; une extension imprimable (SVG JSXGraph) est clonée telle quelle. Langages v1 : `geogebra` (iframe sandboxée `geogebra.org/material/iframe/id/<id>`, id validé `/^[a-zA-Z0-9]+$/` **avant** l'unique `bypassSecurityTrustResourceUrl` — un `[src]` d'iframe l'exige —, non imprimable) et `jsxgraph` (lib npm importée dynamiquement **dans** le composant — double lazy, ~900 kB hors bundle —, équations via **JessieCode** `board.jc.snippet`, jamais eval ; `freeBoard` au re-tracé et au destroy ; imprimable). Config des fences : `clé=valeur` par ligne (`parseExtensionConfig`, doublons ordonnés préservés — plusieurs `equation=`/`point=`). Specs : composant factice en multi-provider dans le TestBed (jsdom exécute `createComponent` sans souci) ; jsxgraph mocké par `vi.mock`. Clés i18n `markdownExtensions.*`. Ajouter un langage = un dossier + une entrée dans `markdown-extensions.providers.ts` **+ sa page de doc** : le champ `doc.loadComponent` de la def est **requis** (l'obligation est portée par le type — import dynamique, jamais statique) et ses clés `docs.pages.<language>.{title,summary}` + prose `docs.<language>.*` (fr/en).

**Export PDF / impression** (`core/print/print.service.ts` + `src/styles/_print.scss`) : l'export PDF passe par l'**impression native du navigateur** (« Enregistrer en PDF ») — zéro dépendance, maths KaTeX **vectorielles** et texte sélectionnable. `PrintService.printCourseContent(source, courseId)` (browser-only, no-op SSR) **clone** le DOM déjà rendu (donc déjà sanitisé par `course-markdown` — on ne re-rend rien), le passe à `transformForPrint`, pose le clone dans un conteneur isolé `#oc-print-root` **enfant direct de `<body>`**, attend le chargement des images (borné `IMG_LOAD_TIMEOUT_MS`, sinon PDF aux images blanches), appelle `window.print()`, et retire le conteneur en `finally`. **Deux consommateurs** : `app-markdown-view` (bouton d'impression **flottant** par bloc — input `showPrint`, défaut `true`, masqué par les hôtes à petits extraits : docs, playground ; méthode publique `print()`) et `course-preview` (bouton « Télécharger en PDF », **cours entier**, `viewChild` `previewContent`). **`transformForPrint`** (pure, testée jsdom) transforme le clone en place : les extensions markdown **non imprimables** (`data-oc-printable ≠ "true"`, ex. iframe GeoGebra) → note « contenu interactif » (les imprimables, SVG JSXGraph, sont clonées telles quelles) ; puis, keyé par `data-oc-resource-id`, les **images restent** (l'URL présignée est encore valide à l'instant de l'impression), l'audio/vidéo → note renvoyant vers l'URL stable, les liens de ressource → `href` réécrit vers l'URL stable, la carte d'un bloc document → nom cliquable + URL en clair copiable. **URL stable** : `resourceContentUrl(courseId, resourceId)` (`resource.utils.ts`) pointe la **gateway publique `/public`** (sans auth, 307 vers l'URL présignée inline S3) — **pérenne**, contrairement à l'URL présignée à TTL court, donc utilisable dans un PDF persistant partagé à des élèves sans compte ; toujours **absolue** via `apiContentBase(apiUrl, siteUrl)` (nouveau champ d'environnement **`siteUrl`**, jamais remplacé par `/api`). Pour que l'export retrouve les ressources, `buildResourceElement` (`course-markdown.ts`) **repose `data-oc-resource-id`** (`RESOURCE_REF_ATTR`) sur chaque élément résolu (inerte à l'écran, survit à DOMPurify via `data-*`). **`keepHeadingsWithContent`** (pure, testée) enveloppe chaque titre du contenu avec les quelques blocs qui le suivent (borné `HEADING_KEEP_FOLLOWING`) dans un `.oc-print__keep` `break-inside: avoid` → pas de titre orphelin en bas de page. Invariants de `_print.scss` : `#oc-print-root` est rendu **hors-écran** (`left:-100vw`) hors impression, **jamais `display:none`** (ses images ne se chargeraient pas avant `print()`) ; en `@media print`, `body > *:not(#oc-print-root)` est masqué, le conteneur **force les tokens de lecture clairs** (texte foncé sur blanc même en thème sombre), `@page { margin: 0 }` supprime les en-têtes/pieds ajoutés par le navigateur (les marges de lecture reviennent par le padding), **un bloc de cours par page** (`break-before: page` sur `.course-preview__block + .course-preview__block`), typographie d'impression en unités **pt**. Clés i18n : `markdownView.print`, `courses.preview.downloadPdf`/`pdfMediaNote`, `markdownExtensions.printFallback` (+ sections `print` des pages de doc).

**Documentation des langages markdown** (`features/docs/` + routes publiques `/:lang/markdown-language/docs[/:slug]`) : une page par « langage » du markdown de cours — intégrés **KaTeX**/**Mermaid** (`features/docs/{katex,mermaid}-doc/`, slugs réservés, déclarés dans `doc-pages.ts`) + une par extension (composant colocalisé `geogebra/geogebra-doc.*`, `jsxgraph/jsxgraph-doc.*`, référencé par `def.doc.loadComponent`) ; `allDocPages(defs)` fusionne intégrés puis registry. `DocsShell` (route `:slug`, **RenderMode.Client** — Monaco + markdown-view ; la route sans slug redirige vers `katex`) : barre d'onglets = **liens router** au motif visuel `.tabs`/`.tab` global + `routerLinkActive` (nav + aria-label, PAS un tablist APG — de vraies navigations) ; corps monté par **`NgComponentOutlet`** (premier usage du repo) sous stale-guard ; slug inconnu → notice (pas de redirect). **Premier composant du projet à survivre à un changement de param de chemin** (onglet→onglet réutilise l'instance) : le paramMap est **observé** (`toSignal(route.paramMap)`), ne pas « simplifier » en snapshot. Chaque page de doc = prose i18n `docs.<slug>.*` + exemples en **constantes TS non traduites** rendus soit statiques (`app-markdown-view` + `<pre class="doc-source">`) soit **modifiables** via `app-markdown-playground` (`shared/markdown-playground/`) : Monaco (`app-markdown-editor`) | aperçu (`app-markdown-view`) côte à côte (grid, empilé <900px), `initial` lu une fois, `FormControl` public `control` (jsdom pilote le contrôle), frappe → aperçu **débouncée 400 ms** (sinon chaque caractère remonterait l'iframe/figure des extensions). `app-markdown-view` expose l'input `showPrint` (défaut true) — les hôtes à petits extraits (docs, playground) masquent le bouton d'impression flottant. Utilitaires globaux `.doc-intro`/`.doc-section`/`.doc-rules`/`.doc-source`/`.doc-more` dans `_components.scss`. `SeoService.apply({titleKey, descriptionKey, path})` généralise `applyHome()` (qui y délègue). Accès : lien « Documentation » public du footer + liens « Documentation complète » par section de la modale d'aide (qui gagne des sections courtes GeoGebra/JSXGraph) — liens de la modale en **`target="_blank" rel="noopener"`** (RouterLink n'intercepte pas : nouvel onglet, contexte d'édition intact, aucune fermeture à câbler) ; tout spec montant la modale (directement ou via markdown-field/exercise-editor) fournit `provideRouter([])`.

**Composants partagés** (`src/app/shared/`) : composants réutilisables transverses. `SubjectPicker` y est le premier `ControlValueAccessor` (valeur = `id` du nœud, utilisable en Reactive Forms) ; tout nouveau form control suit ce modèle (provider `NG_VALUE_ACCESSOR` + `forwardRef`, `writeValue`/`registerOnChange`/`setDisabledState`, navigation clavier + ARIA treeview). `EducationLevelPicker` est le premier CVA **multi**-sélection : valeur = `string[]` d'ids toujours émise en ordre d'arbre (jamais `null`, ids inconnus préservés) ; pattern volontairement plus simple — disclosure + cases à cocher natives (pas de treeview ARIA ni de recherche, arbre ~22 nœuds toujours déplié), chips « badge niveau » (slate) sous le champ. `EducationLevelPicker` accepte un input `systeme` (filtre les **racines** de l'arbre au système scolaire donné ; `null` = tous) — le contrat « ids inconnus préservés » demeure : c'est au parent de vider la valeur quand le système change. `SubjectMultiPicker` (`shared/subject-multi-picker/`) est le CVA multi-matières **par composition** : il embarque le `SubjectPicker` mono (non modifié) comme champ d'ajout via un `FormControl` interne réinitialisé à chaque sélection, valeur = `string[]` en ordre d'ajout dédoublonné, chips « badge matière » (indigo). **Piège de nommage** : `SubjectLevel` et les clés `subjects.level.*` désignent la *profondeur* de l'arbre des matières ; tout ce qui touche aux niveaux d'étude vit sous `EducationLevel*` / `educationLevels.*`. Pièges : un `viewChild()` ne peut pas être posé sur un champ privé ES (`#`) — le marquer `protected` ; et ne pas nommer une variable de template (`#foo`) comme un signal du composant (le template résout la ref avant le signal).

**Notifications globales & spinner** (`core/notifications/` + `shared/snackbar/` + `shared/spinner/`) : `NotificationService` est la variante **in-memory** du patron à signaux (pas de HTTP) — file `toasts` source de vérité, API `error/warning/info/success/dismiss`, ids par compteur incrémental (**jamais** `Date.now()`/`Math.random()`), auto-fermeture par `setTimeout` gardée `isPlatformBrowser` (aucun timer au SSR), dédoublonnage des messages identiques déjà visibles ; les émetteurs poussent un `message` **déjà traduit** (le service et le composant ignorent l'i18n, comme les libellés métier). `Snackbar` (monté **une seule fois** dans `app.html`, ancré bas-gauche, `z-index` élevé, `pointer-events: none` sur le conteneur / `auto` sur les toasts) rend la file : `role="alert"` pour une erreur / `role="status"` sinon, liseré gauche de la teinte **500** sémantique + glyphe distinctif (l'info n'est **jamais** encodée par la seule couleur, et 500 reste un aplat, jamais du texte). Câblage actuel : **seul `AuthService`** l'alimente (toast `loginError` quand la discovery échoue au login, `connectionError` quand le refresh silencieux échoue au bootstrap) — `AuthService` reste seul importeur d'`angular-oauth2-oidc`, il ne fait qu'appeler ce service applicatif ; brancher les autres sources (HTTP, données) plus tard sans refonte. `Spinner` (design system, `role="status"` + label `.sr-only` « Chargement… » par défaut, tailles `sm/md/lg`, anneau CSS coloré par `--color-primary`, rotation figée en `prefers-reduced-motion` → repli statique) recouvre l'éditeur Monaco tant que `ready()` (ex-`#ready`, exposé en lecture) est faux dans le markdown-editor. Les clés vivent sous `notifications.*` et `common.loading` (fr/en symétriques) ; l'utilitaire `.sr-only` est global (`_base.scss`). Doc UI : §8 de `DESIGN_SYSTEM.md`.

**Chaîne i18n (fragile, ne pas "simplifier")** — Transloco fr/en :
- Le loader (`src/app/core/i18n/transloco-loader.ts`) passe par des `import()` dynamiques de JSON, **pas** par HTTP : au prerender il n'y a aucun serveur pour répondre à une URL relative.
- Un `provideAppInitializer` dans `app.config.ts` précharge la langue active avant le premier rendu — sans lui, le HTML prerendered sort avec des chaînes vides.
- La langue persistée de l'utilisateur est restaurée **après** l'hydratation (`afterNextRender` dans `App`) : la restaurer plus tôt fait diverger le premier rendu client du DOM serveur (erreur NG0500).
- Traductions dans `src/app/i18n/{fr,en}.json` — garder les deux fichiers symétriques.

**Thème clair/sombre** : attribut `data-theme="dark"` sur `<html>`, posé avant le premier paint par le script inline d'`index.html` (anti-FOUC). `ThemeService` **lit** cet attribut au bootstrap au lieu de recalculer — préserver ce contrat. Les swaps visuels dépendant du thème (ex. logo) se font en CSS sur `[data-theme]`, jamais par binding (mismatch d'hydratation).

**OIDC** : `AuthService` (`src/app/core/auth/`) est la **seule** couche autorisée à importer `angular-oauth2-oidc` (exigence de remplaçabilité de l'IdP, Descriptions.md §8). Code flow + PKCE, client public. Init browser-only et sans appel réseau au démarrage : les pages publiques élèves ne doivent jamais dépendre de la disponibilité de Zitadel. Le `state` restauré au callback est validé (chemins internes uniquement).

**Onboarding bloquant** (`features/onboarding/`, route `:lang/onboarding`, RenderMode.Client) : au retour du callback (`AuthCallback`), le profil est chargé (`ensureLoaded()`) et un profil incomplet est redirigé vers `/:lang/onboarding?next=<cible>`. Les routes protégées portent `canActivate: [authGuard, onboardingGuard]` — **dans cet ordre** (le guard onboarding laisse passer les non-authentifiés), et `onboardingGuard` ne doit **jamais** être posé sur la route onboarding elle-même (boucle) : c'est le composant qui renvoie un profil déjà complet vers `next`. Guard et callback sont **fail-open** si l'API profil est injoignable (ne jamais enfermer l'utilisateur hors de l'app). La page est le premier usage des **Reactive Forms typés** du projet : en zoneless, la réactivité du template passe par `toSignal(form.valueChanges)` ; le stepper est **dynamique** (liste d'étapes `computed` dérivée des rôles cochés — prof, élève ou les deux ; décocher un rôle vide son bloc, changer de système vide les niveaux choisis).

**Page profil** (`features/profile/`, route `:lang/profile`, `canActivate: [authGuard, onboardingGuard]`, RenderMode.Client) : mêmes sections que l'onboarding à plat, pré-remplies via `ensureLoaded()` + `patchFormFromProfile`, sauvegarde par `saveProfile()`. « Enregistrer » n'est actif que si le formulaire est **complet ET modifié** — détection par snapshot JSON du payload (déterministe : niveaux en ordre d'arbre, matières en ordre d'ajout). **Menu utilisateur** (`layout/header/user-menu.*`) : pattern APG menu-button (déclencheur = nom de l'utilisateur, `aria-haspopup="menu"`, panneau `role="menu"`, Escape → refocus déclencheur, focusout externe → ferme, flèches ouvrent/cyclent) ; « Se déconnecter » vit dans ce menu, plus dans le header. Zoneless : le focus post-ouverture passe par `afterNextRender(..., { injector })` car le panneau n'est pas encore rendu au `set(true)`.

**Environnements** : `environment.ts` (défaut dev) remplacé par fileReplacements (`.development.ts` / `.prod.ts`). Le type commun vit dans `environment.model.ts`, qui n'est **jamais** remplacé — ne pas importer de types depuis `./environment` dans les fichiers remplacés (import circulaire au build). Config figée au build (pas d'injection runtime), sauf `ALLOWED_HOSTS` / `TRUST_PROXY_HEADERS` / `PORT`, lues par `src/server.ts` (protection SSRF d'Angular SSR : sans `ALLOWED_HOSTS` correct, le serveur répond 400).

**Design system** : les tokens CSS de `src/styles/_tokens.scss` recopient le §10 de `DESIGN_SYSTEM.md` — toute couleur/typo passe par `var(--…)`, jamais de hex en dur dans les composants. Cibles de contraste : AA partout, AAA dans `.course-content`. Pièges connus : `slate-400` interdit pour du texte (réservé désactivé/décoratif), l'ambre n'est jamais une couleur de texte, texte blanc sur indigo uniquement sur indigo-600/700. Polices auto-hébergées (@fontsource, déclarées dans `angular.json > styles` — pas d'`@import` Google Fonts). Libellés en sentence case (§9). **Glisser-déposer** : la poignée `.drag-handle` (icône `grip-vertical`) **et** les états visuels CDK (`.cdk-drag-preview`/`.cdk-drag-placeholder`/`.cdk-drag-animating`) sont des **utilitaires globaux** dans `_components.scss` (motif `.tabs`/`.pill` ; l'aperçu flottant est en plus détaché dans `<body>`) ; la poignée est un affordance porteur de sens → `--text-secondary` (slate-500, 3:1) au repos / `--text-primary` au survol, **jamais** slate-400 ; les flèches ↑/↓ conservées sont atténuées au repos, révélées au survol/focus. `@angular/cdk` est une dépendance **directe** (drag-drop), pas seulement transitive de Monaco. **Médias de ressources intégrés au markdown** (`.course-content img`/`audio`/`video` dans `_tokens.scss`) : contraints à la largeur de lecture (`max-width:100%`, `height:auto`, `border-radius` sur l'image), le rythme vertical venant du `<p>` qui les enveloppe ; `.course-resource--missing` = note atténuée (bordure dashed, `--text-secondary`, jamais slate-400) affichée à la place d'une ressource supprimée/indisponible.

**Sass** : `styles.scss` n'utilise que `@use` (jamais `@import` scss, déprécié — et `@use` doit précéder toute autre règle).
